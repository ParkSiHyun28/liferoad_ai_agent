# 시연 시나리오 — My LifeRoad

데모 기준일: 2026-10-03

---

## 개요

"묻기 전에 먼저 알림"이라는 핵심 가치를 두 페르소나로 보여주는 약 2~3분 분량 시연이다.
흐름은 능동 점검 → 자유 대화 순서로 진행한다.

---

## 페르소나 소개

| 이름 | 국적 | 체류 자격 | 주요 관심사 |
|---|---|---|---|
| 응웬 반 민 (minh) | 베트남 | E-9 근로 | 연금 반환 / 보험 / 본국 송금 |
| 수만 라이 (suman) | 네팔 | D-2 유학 | 예치금 담보대출 / 신용 구축 / 비자 갱신 |

---

## 1단계: 능동 점검 (Proactive Check)

### 진행 방법
1. 사이드바에서 페르소나를 선택한다.
2. "능동 점검 실행" 버튼을 누른다.
3. 에이전트가 묻지 않아도 먼저 위험과 마감을 찾아 카드로 띄운다.

### 민(minh)에게 뜨는 카드 예시
- 출국만기보험 반환 신청 마감이 출국 후 14일 이내임을 경고한다.
- 베트남 송금 환율 우대 창이 이번 주 닫힘을 알린다.
- 건강보험 지역가입 전환 서류 누락 항목을 짚어준다.

### 수만(suman)에게 뜨는 카드 예시
- 전세 예치금으로 받을 수 있는 담보대출 한도를 먼저 계산해서 보여준다.
- 신용 이력 0개월 상태이므로 이번 달 안에 신용카드 신청을 권고한다.
- 외국인등록증 만료가 45일 이내임을 알리고 갱신 절차를 안내한다.
- 체류 기간 중 발생한 미신고 아르바이트 수입이 체류 조건을 위반할 수 있다고 경고한다.

### 강조 포인트
능동 점검은 사용자가 아무 질문도 하지 않았는데 에이전트가 먼저 금융 손실과 마감을 찾아주는 기능이다.
"챗봇이 아니라 동반자"라는 메시지를 전달한다.

---

## 2단계: 자유 대화 (Free Chat)

능동 점검 카드를 확인한 뒤 대화창에서 아래 질문을 이어서 입력한다.
각 질문은 특정 tool 호출을 확실히 유발하도록 설계했다.

### 민(minh) 추천 질문

1. "민 씨 연금 얼마 받아요?"
   - 호출 tool: `pension_estimator`
   - 기대 응답: 가입 기간과 납입액 기반 예상 반환일시금 금액

2. "송금 수수료 아끼는 법 알려줘"
   - 호출 tool: `remit_optimizer`
   - 기대 응답: 환율 우대 시간대와 수수료 비교 결과

3. "출국만기보험 언제까지 신청해요?"
   - 호출 tool: `deadline_radar`
   - 기대 응답: 출국 예정일 기준 신청 마감일과 필요 서류 목록

4. "반환일시금 신청서 작성해줘"
   - 호출 tool: `form_autofill`
   - 기대 응답: 페르소나 정보가 채워진 신청서 초안

### 수만(suman) 추천 질문

1. "수만 씨 예치금으로 대출 얼마 돼요?"
   - 호출 tool: `collateral_calc`
   - 기대 응답: 예치금 담보 인정 비율과 한도 계산 결과

2. "신용 어떻게 쌓아요?"
   - 호출 tool: `credit_builder`
   - 기대 응답: 외국인 신용 구축 단계별 플랜(통장 개설 → 체크카드 → 신용카드 순)

3. "유학생 아르바이트 몇 시간 가능해요?"
   - 호출 tool: `compliance_reason`
   - 기대 응답: D-2 체류 조건상 허용 근무 시간과 위반 시 불이익 안내

4. "외국인등록증 갱신서 작성해줘"
   - 호출 tool: `form_autofill`
   - 기대 응답: 페르소나 정보가 채워진 외국인등록증 연장 신청서 초안

## 녹화 팁

- 화면 오른쪽 하단의 "처리 과정" 패널을 반드시 펼쳐둔다. LLM이 어떤 tool을 언제 호출했는지 실시간으로 보여야 "AI가 실제로 작동하고 있다"는 신뢰감을 준다.
- 능동 점검 카드가 뜨는 순간을 천천히 보여준다. 3~4초 멈춰서 심사위원이 읽을 시간을 준다.
- 자유 대화는 질문 타이핑 후 응답 전까지 0.5~1초 정도 멈추면 자연스럽다.
- 전체 분량은 능동 점검 40초 + 민 대화 50초 + 수만 대화 50초 = 약 2분 20초가 목표다.
- 두 페르소나를 모두 보여줄 시간이 부족하면 민(minh) 시나리오를 우선한다. 연금 반환과 송금이 외국인 근로자 금융 고충을 가장 직관적으로 보여주기 때문이다.
